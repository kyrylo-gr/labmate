# Version 0.11.0 — lifecycle hooks and mirroring

## Motivation

Labmate’s acquisition flow is built around a canonical on-disk layout: an HDF5 file plus optional sidecar files in the experiment directory. In practice, that layout creates two recurring needs.

First, **redundancy and portability**: teams often want every saved acquisition copied to a second root (backup volume, shared storage, or a machine-local cache) without changing how notebooks call `save_acquisition`.

Second, **analysis when the primary tree is incomplete**: the notebook may reference a path under the data directory while the `.h5` is missing locally. Analysis should be able to **recover the file** from a mirror using the same logical path.

## Implementation

1. **`LifecycleHooks`** ([`labmate/acquisition/hooks.py`](../../labmate/acquisition/hooks.py)) holds named callback lists. Managers call **`dispatch_*`** at fixed lifecycle points; callbacks run **synchronously** on the caller’s thread. Plugins that need background I/O (e.g. the documented **`MirrorDirectoryPlugin`** example in [`advanced_examples.md`](../starting_guide/advanced_examples.md)) can use their own **`ThreadPoolExecutor`** inside the callback.

2. **`AcquisitionManager`** exposes **`hooks`** (optional shared instance via constructor). It dispatches **`dispatch_acquisition_data_loaded`** after building a `NotebookAcquisitionData` and **`dispatch_acquisition_saved`** after `save_acquisition`.

3. **`AcquisitionAnalysisManager`** passes **`hooks`** through to the base manager, dispatches **`dispatch_figure_saved`** after `save_fig`, **`dispatch_analysis_data_loading(full_h5)`** on every `analysis_cell` before opening the file, **`dispatch_acquisition_cell_ready`** on acquisition step 1, and **`dispatch_analysis_cell_ready`** after load/lint. **`set_acquisition_cell_prerun_hook`** / **`set_analysis_cell_after_load_hook`** delegate to **`set_acquisition_cell_ready`** / **`set_analysis_cell_ready`** on `hooks`.

4. **`MirrorDirectoryPlugin`** is documented as **example code** in **[`docs/starting_guide/advanced_examples.md`](../starting_guide/advanced_examples.md)** and **`docs/examples/more/use_local_mirror_backend.ipynb`** (not part of the installable package). It registers mirror-on-save on **`add_acquisition_saved`** and **`add_figure_saved`**, and restore-before-load on **`add_analysis_data_loading`**. Attach with **`MirrorDirectoryPlugin(root).attach(aqm.hooks)`**.

5. End-user reference: **[`docs/code/lifecycle_hooks.md`](../code/lifecycle_hooks.md)**. Examples: **[`docs/starting_guide/advanced_examples.md`](../starting_guide/advanced_examples.md)**, notebook **`docs/examples/more/use_local_mirror_backend.ipynb`**.

6. **Tests:** [`tests/acquisition_tests/hooks_test.py`](../../tests/acquisition_tests/hooks_test.py) covers hook dispatch.
